Before we talk

What I need from you — and how I work

I ask for a full project description and the complete scope of work — in writing — before any phone call. It saves us both time and makes the quote accurate and efficient: I come to the conversation already understanding your project, with a real plan and a flat price, instead of a vague cold call. Here's exactly what to send, and how I work.

Step 1

Start here — what to put in your first email

Email these to scottbaker@rusticagentic.agency. The more detail, the faster and more accurate my quote.

  • What you need — built or fixed, and the goal behind it
  • Your current site or system + a link, if there is one
  • Timeline — any deadline you're working toward
  • Budget range — even rough; it helps me scope the right solution
  • Platform/stack if you know it — WordPress, PHP/Laravel, React, AWS, data/Databricks
  • Any compliance needs — e.g. HIPAA

I reply with a flat price and a plan, usually the same day. If a call makes sense, we book it after I've reviewed — never before.

Step 2 · Access

The access I need to do the work — by platform

The rule is the same everywhere: scoped, least-privilege, revocable access, granted only once you've approved the quote and we start — never up front, and never your root/owner password. You can revoke everything the moment we're done.

WordPressAn Administrator WordPress account, plus hosting control-panel access (cPanel/Plesk) or SFTP/FTP. Database access only if the job needs it. Domain/DNS registrar access only if we're changing DNS.
Web hosting & migrationsLogin to the current host and the new host (control panel or SSH), plus domain-registrar access if DNS has to move. I migrate from my machine with zero downtime.
AWSA scoped IAM user or cross-account IAM role with least-privilege permissions for the task — console and/or an access key for the CLI. Never the root account. Billing stays on your account, and you can detach my access instantly.
Databricks & dataA scoped Personal Access Token (or service principal) with the workspace permissions the job needs, plus access to the relevant data sources (e.g. a read-scoped storage/S3 credential) and the cluster or SQL warehouse I'll run on. You own the workspace and its costs.
Custom apps (PHP/Laravel, React, static)A scoped SSH / deploy user or hosting-panel access, plus the Git repository and any environment/deployment access required. No shared owner credentials.
Good to know

How I work

  • Your machine stays yours. I work entirely from my own computer — you grant the access needed and revoke it when we're done. You never share root or personal passwords.
  • Scoped, revocable access only. For AWS that means a limited IAM account, never your root login.
  • Flat quotes, no hourly meter. You approve a fixed price before I touch anything.
  • You pay infrastructure costs directly — hosting, AWS, Databricks — to those providers. My fee is labor only, so there are no surprise bills from me.
  • Technical work only. I don't do graphic design, logos, or custom artwork of any kind — I work with clean placeholders or any branding/assets you already have.
  • Calls come after I've reviewed your written requirements — never a cold call.
